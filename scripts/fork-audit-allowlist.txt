# scripts/fork-audit-allowlist.txt
#
# fgrep-safe substrings that the audit-fork-drift.sh script should ignore.
# Each entry is matched literally against the `filename:lineno:content` lines
# emitted by `grep -rn`. Lines are filtered out via `grep -vFf`.
#
# Convention: every entry is preceded by a `# <path>: why kept` comment so
# triage decisions stay auditable. When the underlying drift gets ported in
# a later phase (e.g. invest-research YT flow lands in K2Bi at Phase 2),
# remove the corresponding allowlist entry rather than leaving it stale.
#
# Avoid whole-file allowlists for live-developed skills: a `invest-X/SKILL.md`
# substring suppresses every future K2B reference added to that file too,
# defeating the audit's purpose. Prefer line-specific patterns even when the
# entry count grows.
#
# Scoping limitation: `audit-fork-drift.sh` filters via `grep -vFf`, which
# applies every substring globally to the `filename:lineno:content` output.
# A substring listed here will suppress hits in ANY file that contains it,
# not just the file the comment names. Mitigation: prefer substrings that are
# unique to their intended host file (specific helper-script paths, distinctive
# prose phrases) rather than short generic tokens. If a substring must be
# generic, accept the cross-file false-suppress radius and consider it on the
# next /ship's audit pass. A future audit-script upgrade could carry per-file
# sections, but the fgrep-based allowlist is the architect's current contract.
